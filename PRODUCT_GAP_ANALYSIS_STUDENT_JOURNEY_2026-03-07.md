# Product Gap Analysis — International Student Job-Seeker Journey

Date: 2026-03-07
Context: "If I were an international student looking for H1B jobs, what would I think when opening h1bfinder.com, and what needs are unmet?"

## 1) What users think on first visit

- Is this data trustworthy and up-to-date?
- Can this help me make decisions right now, or is it just a dashboard?
- Which companies/roles should *I* apply to based on my profile?
- What is my next concrete action after reading this page?

## 2) Core unmet needs (the gap)

### Gap A — Data view vs Personal decision support
Current: aggregate rankings and trend data.
Missing: profile-aware recommendations.

User need:
- Input profile (major, role preference, location, experience)
- Output personalized target company/title list

### Gap B — Insight vs Action plan
Current: users can explore numbers.
Missing: users are not told what to do next.

User need:
- Prioritized application list
- Weekly application plan
- Suggested role/title targeting strategy

### Gap C — Volume metrics vs Risk/quality metrics
Current: filings/approvals/salary are visible.
Missing: stability and risk indicators.

User need:
- Sponsor stability score (multi-year consistency)
- Trend volatility warning
- “High volume but unstable” signal

### Gap D — Broad title categories vs practical role granularity
Current: titles exist but practical decision guidance is weak.
Missing: role-level conversion hints.

User need:
- Distinguish SWE/ML/Data/PM subtracks
- Better title normalization and role grouping

### Gap E — Historical data vs current-cycle strategy
Current: historic trend shown.
Missing: timeline strategy for current hiring cycle.

User need:
- “When to apply” guidance
- Seasonality hints
- Timing recommendation by role/company type

### Gap F — Information product vs job-outcome tool
Current: analytics product experience.
Missing: outcome-driving tools.

User need:
- Resume keyword guidance for sponsor-heavy roles
- JD match helper
- Interview prep direction linked to target sponsor set

## 3) Positioning mismatch (root problem)

Current product impression:
- “H1B analytics dashboard”

Target user expectation:
- “Give me my best path to an H1B-offer outcome”

## 4) Priority fixes (2-week feasible MVP)

### Priority 1 (must-have)
- Add “Personalized Plan” entry on homepage
- Input form: major + target role + target city + years experience
- Output card:
  - Top 20 target sponsors
  - Recommended titles
  - Weekly action checklist

### Priority 2
- Add sponsor stability score + volatility badge
- Add CTA below every chart: “Generate my action plan”

### Priority 3
- Add explainable recommendation panel
  - “Why this company/title is recommended”
  - data-backed factors used

## 5) Success metrics for this gap closure

- % users starting personalized flow
- % users who export/save plan
- % users returning within 7 days
- % chat sessions ending with actionable click (companies/titles/plan)

## 6) Working hypothesis

If h1bfinder shifts from “data exploration” to “personalized action planning,”
conversion and retention should improve because users can immediately act on recommendations.
